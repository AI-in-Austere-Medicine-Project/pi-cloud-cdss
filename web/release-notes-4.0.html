Headline: the whole system fits on a desk

Version 4.0 moves EdgeCDSS off rented cloud infrastructure entirely. The guideline knowledge base, retrieval engine, deterministic safety gates, web interface, structured feedback system, and audit logs all run on a single NVIDIA Jetson Orin Nano at the point of care. Remote access flows through an outbound-only encrypted tunnel — no open ports, no exposed addresses, no cloud vendor between a provider and the guidance. The deployment is network agnostic: Starlink, broadband, Wi-Fi, Ethernet, or LTE/5G.

Knowledge → Logic → AI → Validation → Human

Deterministic Python code performs medication calculations, contraindication checks, routing, patient-context handling, and safety gates. AI is restricted to language generation, retrieval support, and semantic validation.

What's new in 4.0

Deterministic-first architecture

  • Every stated dose is computed by code, not AI. Doses are calculated in Python from confirmed patient weight and drug concentration, then passed to the language model as a fixed ALLOWED_DOSES contract. The model formats language; it is prohibited from doing medication math.
  • Thirteen pre-generation safety gates decide missing-weight, missing-route, pediatric limits, and absolute contraindications in code, before any AI is consulted — many queries resolve in milliseconds without an AI call.
  • Deterministic dose verification. Generated responses are parsed and every stated dose is checked against the computed contract. A response that states a dose the code did not calculate is blocked.
  • Fail-closed validation. A narrow semantic validator reviews every draft; failures produce a safety hold, never a corrected guess. Structured false-positive overrides — written in code from observed field reports — keep the gate honest without weakening it.

Edge self-hosting

  • Full stack migrated from a cloud VM to the Jetson Orin Nano after a provider capacity outage took the previous deployment offline.
  • On-device retrieval: 89 JTS CPGs ingested into 8,559 passages with sentence-aware chunking, header/footer stripping, and page-accurate citations; embeddings computed locally at zero per-query cost.
  • Clinical router: an LLM-built protocol index maps field language to the correct CPG and sharpens retrieval before it runs.
  • Services are systemd-managed and survive power loss; the tunnel reconnects automatically on any network.

Field evaluation loop

  • Structured clinical feedback on every response: one tap for "clinically appropriate," or a flag with severity (minor / significant / dangerous-if-followed), issue categories, and a protocol-cited correction.
  • Session audit logging (no PHI) makes reported failures reproducible; a beta-reported wrongful safety block was diagnosed from the log and fixed, with a regression test, the same day.
  • Automated 24-case clinical suite — pediatric gates, P1 safety blocks, RSI protocols, grounded scenarios — passes 24/24 against the live endpoint, up from 61.8% two versions ago.
  • Patient context now persists across the entire conversation: durable facts (weight, age, route, access) hold for the full encounter, while clinical events are windowed to recent turns by design.
  • Voice responses speak clinical notation correctly — "0.24 mL of 100mg/mL" is pronounced as words, with routes and acronyms spelled out.

Cost profile

Compute is a one-time ~$249 (plus ~$50 storage). Recurring cost is dominated by connectivity — roughly $120–150/month for satellite service where used — while language-model API usage runs in single-digit dollars per month at evaluation volume. The AI is nearly free; the pipes are not.

Known boundaries

EdgeCDSS is decision support, not a decision-maker. It is a research prototype evaluated with simulated and synthetic scenarios only — not validated for clinical use and not for patient care decisions. Language generation currently requires connectivity to a cloud model; the generation layer is deliberately swappable, and a cross-model comparison (OpenAI / Claude / Gemini) is planned. A fully offline on-device model remains the research goal (Project 02).
